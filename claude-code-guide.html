PROMPT 01 — 기획 시작
폴더 안의 Sample_PORTFOLIO_TEMPLATE_GUIDE.md 를 읽고, 동일한 구조와 디자인으로 내 포트폴리오를 만들어줘.
내 자료는 폴더 안에 있어. 먼저 어떤 섹션 구성과 컬러 방향으로 만들지 제안해줘.
PROMPT 02 — 내 정보 반영
가이드의 디자인 시스템(CSS 토큰, 섹션 순서)은 그대로 유지하고,
내 경력과 이미지 파일들을 적용해서 단일 HTML 파일로 만들어줘.
이미지는 capture/g/ 폴더에 있어.
PROMPT 03 — 수정 요청
HERO 섹션의 수치를 [실적 수치]로 바꾸고,
GALLERY 썸네일은 gal-01.jpg ~ gal-04.jpg 4장으로 구성해줘.
TIP  프롬프트는 순서대로 — 기획 제안 받기 → 내 정보 반영 → 세부 수정 순으로 진행

# 1. 협업 체계
나는 비개발자로서 기획·방향을 총괄합니다. 내 지시와 AI 아키텍트 설계서를 병렬 수용하여 실무 코드로 구현하세요.
 
# 2. 임의 판단 및 과시용 코딩 금지
- 임의 확장 금지: 요청하지 않은 기능 추가 절대 금지.
- 껍데기 코딩 금지: 로직을 주석(// 여기에 구현)으로 날림 처리 금지.
- 독단적 파일 수정 금지: 연관 파일 수정 전 반드시 보고 후 승인.
 
# 3. UI/UX 표준
- 색상: 디자인 토큰만 사용, 인라인 하드코딩 금지.
- 접근성: 11px 미만 폰트 금지.
- 클릭 영역: 버튼·링크 최소 44px 확보.
 
# 4. 작업 진행 원칙
- 큰 작업 전 "이렇게 진행할게요, 맞나요?" 확인 후 시작.
- 한 번에 하나씩: 잘게 쪼개 단계별 진행.
- 오류 시 전체 재작성 금지 → 오류 메시지 보고 후 부분 수정.
 
# 5. 모델 및 세션 관리
- 기본 모델: Sonnet 사용 (복잡한 설계만 Opus).
- 메모리 80% 이상 시: claude.md 저장 후 새 창에서 계속.
- 지시사항은 .md 파일로 폴더에 미리 정리해서 제공.
 
# 6. 검증 위주 보고 원칙
완료 시 반드시 아래 3가지 보고:
① 수정·신설 파일 경로 목록

③ 연관 파일 영향 위험 요인 고지

APPENDIX A5
GitHub — 안전하게 보관하기
코드와 파일을 클라우드에 저장해두는 서비스 — 컴퓨터가 망가져도 안전하고, 어디서든 로그인만 하면 이어서 작업할 수 있다
1
github.com 가입
이메일만 있으면 충분
2
저장소(Repository)* 하나 만들기
"New" 버튼 → 이름 정하고 Create (포트폴리오 자료 넣을 "방" 하나 만드는 것)
3
Claude Code에게 요청
"이 폴더를 GitHub에 올려줘" — 커밋(Commit)* 명령어는 외울 필요 없음
흔히 하는 실수
공개(Public)*·비공개(Private)* 구분 안 하기 → 개인정보·API 키 있으면 비공개로
커밋 없이 며칠치 작업 몰아서 하기 → 중간중간 저장해야 되돌릴 지점이 생김
터미널 명령어를 직접 외워서 치려다 막힘 → Claude Code에게 시키면 인증까지 대신 처리
커밋 메시지 의미 없이 적기 → "수정함"(X) / "포트폴리오 사진 갤러리 추가"(O)
* 저장소(Repository) — 파일을 모아두는 공간  ·  커밋(Commit) — 저장 시점을 기록하는 것  ·  공개/비공개(Public/Private) — 누구나 볼 수 있는지 여부
핵심  명령어를 외울 필요 없다 — Claude Code에게 시키면 저장부터 업로드까지 대신 처리한다
APPENDIX A6
Vercel — 가장 쉬운 배포 방법
만든 HTML을 실제 인터넷 주소(URL)로 띄워주는 서비스 — 포트폴리오는 결국 링크로 보여줘야 한다. GitHub과 연결해두면 커밋할 때마다 자동으로 다시 배포(Deploy)*
1
vercel.com → GitHub로 가입
"Continue with GitHub" — 계정 새로 안 만들어도 됨
2
가져오기(Import)*에서 저장소 선택
방금 만든 GitHub 저장소 고르기
3
Deploy 버튼 한 번
몇 초 후 xxx.vercel.app 주소 생성됨
흔히 하는 실수
수정했는데 그대로 보임 → 재배포됐는지, 새로고침했는지 확인
이미지가 깨져서 보임 → 파일 경로 대소문자·폴더 위치 확인
도메인 헷갈림 → 기본 제공 주소로 충분, 따로 살 필요 없음
무료 한도 걱정 → 개인 포트폴리오 트래픽으로는 거의 도달 안 함
* 배포(Deploy) — 만든 파일을 실제로 인터넷에 올리는 것  ·  가져오기(Import) — GitHub 저장소를 Vercel로 연결하는 것
핵심  GitHub에 올려져 있으면 Vercel 연결은 클릭 몇 번이면 끝난다
